# CLAUDE.md (Global)

이 파일은 모든 프로젝트에서 나를 지원하기 위한 기본 개발 가이드이다.

---

## 1. 개발자 프로필

- 백엔드 중심 풀스택 개발자 (Java/Spring 기반)
- 경력 성장 목표: 3~5년차 이상 레벨의 문제 해결 역량 확보
- 관심 분야:
  - 인프라 구조 설계
  - 성능 최적화
  - 데이터 처리 효율화
  - 문서 자동화 및 코드 생성

---

## 2. 작업 환경

- 회사 PC: Windows
- 개인 PC: MacBook
- IDE: IntelliJ IDEA (주력), VS Code, Eclipse
- DB Tool: DBeaver

---

## 6. 문제 해결 시 우선 고려 사항

문제를 제시받으면 다음 순서로 접근한다:

1. 인프라 레벨 문제 가능성
2. DB 쿼리 비용, 성능 및 인덱스 문제
3. 트랜잭션/동시성 문제
4. 동시성/락 문제
5. 코드 구조 문제
6. 단순 구현 오류

---

## 7. 성능 중심 개발 원칙

기능 구현 시 항상 다음을 기본 전제로 고려한다:

### 1) DB 레벨
- 실행 계획 확인
- 인덱스 전략 검토
- N+1 문제 차단
- 불필요한 join 제거
- 대량 데이터 시 페이징/커서 기반 처리
- count 쿼리 비용 고려

### 2) 애플리케이션 레벨
- 객체 생성 최소화
- 불필요한 컬렉션 복사 제거
- Stream 남용 지양 (대량 처리 시)
- GC 부담 고려
- 동시성 이슈 검토

### 3) 인프라 레벨
- 네트워크 호출 횟수 최소화
- 캐싱 전략 고려 (로컬/Redis)
- 배치 처리로 전환 가능성 검토
- 수평 확장 가능한 구조 설계

### 4) 트래픽 가정
- 단건 기준이 아닌 “대량 트래픽 기준”으로 사고
- 10배 트래픽 증가 시 병목 지점 예측
- 장애 발생 가능 지점 사전 분석
