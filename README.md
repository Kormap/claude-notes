# 📘 CLAUDE Global Development Guide

> 개인 개발 표준을 정의하는 `CLAUDE.md` 버전 관리 저장소  
> LLM 기반 개발 지원을 위한 운영 가이드 관리 목적

---

# 🎯 목적

- 모든 프로젝트에서 일관된 개발 지원 기준 유지
- 인프라 / 성능 / 데이터 중심 사고방식 고정
- LLM 응답 품질 표준화
- 반복 설계 방향 자동화 자산화

---

# 🧠 설계 철학

- 기능보다 구조를 먼저 설계
- 성능은 1급 요구사항
- 단순하지만 확장 가능한 구조
- 운영 가능한 코드 지향
- 대량 트래픽 기준 사고

# 🚀 활용 방식

## 1️⃣ 글로벌 기준 파일로 사용

각 프로젝트 루트에 심볼릭 링크 또는 복사하여 사용합니다.

```bash
ln -s ~/claude-global/CLAUDE.md ./CLAUDE.md
```

## 2️⃣ LLM 응답 품질 통제

`CLAUDE.md`는 다음 사고 흐름을 강제합니다:

- 인프라 → DB → 트랜잭션 → 코드 구조 순 접근
- N+1, 인덱스, 실행 계획 기본 검토
- 트래픽 증가 시 병목 지점 예측
- 단순 구현 오류보다 구조 문제 우선 점검

---

## 3️⃣ 자동화 확장

이 저장소는 단순 문서 보관을 넘어서 다음 확장을 고려합니다:

- SQL 템플릿 자동 생성
- MyBatis XML 제네레이터
- DTO 매핑 코드 생성
- 테스트 코드 스캐폴딩
- API 문서 자동화 스크립트
- CI 기반 프롬프트 검증

---

## 🏗 운영 전략

### 🔹 인프라 관점

- 네트워크 호출 최소화
- 캐싱 전략 전제 설계
- 수평 확장 가능한 구조 유지
- 배치 전환 가능성 고려

### 🔹 DB 관점

- 실행 계획 검토 필수
- 인덱스 전략 우선 검토
- 대량 데이터 시 Offset Pagination 지양
- Count 쿼리 비용 고려

### 🔹 애플리케이션 관점

- 불필요한 객체 생성 제거
- Stream 남용 방지
- GC 부담 예측
- 동시성 이슈 사전 점검

---

## 📈 성장 목적

- 3~5년차 이상 수준의 문제 해결 사고 체계 고도화
- 운영 관점 개발 역량 강화
- 구조 설계 중심 사고 훈련
- LLM을 생산성 도구가 아닌 설계 보조 엔진으로 활용
